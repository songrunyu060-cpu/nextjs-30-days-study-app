# Day 22-23：图像与字体优化 — `next/image`、`next/font` 与 LCP

## 学习目标

- 掌握 **`next/image`** 的懒加载、尺寸与 `priority` 对 **LCP** 的影响。
- 理解 **`next/font/google`** 与 CSS 变量结合，减少 **FOIT/FOUT** 与布局抖动。
- 对照本项目 **`layout.tsx`** 与 **Tailwind v4** 的全局样式。

---

## 与本项目真实配置

### 1. 字体（已在用）

`src/app/layout.tsx`：

- **`Geist`** 与 **`Geist_Mono`** 来自 `next/font/google`；
- `variable: "--font-geist-sans"` 等注入 CSS 变量；
- `body` 使用 `className={...variable... antialiased bg-background text-foreground}`。

这与 **shadcn** 推荐的「字体变量 + Tailwind」一致。

**练习**：在 DevTools **Network** 里观察字体子集是否按需加载；检查 **`subsets: ["latin"]`** 是否覆盖你的用户语言（若需中文，评估补充字体源）。

### 2. 图像（按需引入）

当前首页/图书页若使用 **`<img>`**，可逐步替换为 **`next/image`**：

- 提供 **`width` / `height`** 或 **`fill` + 父级 `relative`**，避免 CLS；
- 对 **首屏主视觉** 使用 **`priority`**；
- **远程域名**需在 `next.config.ts` 配置 `images.remotePatterns`（本项目尚未配置时，远程图会报错）。

## 案例代码 ①：本仓库 `layout.tsx` —— `next/font` + Tailwind 令牌

```tsx
import { Geist, Geist_Mono } from "next/font/google";

const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
});

const geistMono = Geist_Mono({
  variable: "--font-geist-mono",
  subsets: ["latin"],
});

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body
        className={`${geistSans.variable} ${geistMono.variable} antialiased bg-background text-foreground`}
      >
        {children}
      </body>
    </html>
  );
}
```

在 Tailwind 中可用 `font-sans`（若 `globals.css` 已把 `--font-geist-sans` 绑到 `font-family`）。

## 案例代码 ②：`next/image` —— 本地 + 远程

**本地 `public/cover.jpg`**：

```tsx
import Image from "next/image";

export function BookCover() {
  return (
    <Image
      src="/cover.jpg"
      alt="封面"
      width={320}
      height={480}
      className="rounded-md object-cover"
      priority
    />
  );
}
```

**远程图（需在 `next.config.ts` 声明域名）**：

```ts
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: "https",
        hostname: "images.example.com",
        pathname: "/covers/**",
      },
    ],
  },
};

export default nextConfig;
```

```tsx
<Image
  src="https://images.example.com/covers/abc.jpg"
  alt="封面"
  width={320}
  height={480}
/>
```

## 1. LCP（Largest Contentful Paint）与图片

- 常见 LCP 元素：**大图、标题块、视频海报**。
- 优化顺序：**缩小资源** → **合适格式（AVIF/WebP）** → **`priority`** → **CDN**（Vercel Image Optimization 自动处理需配置允许域名）。

---

## 2. 与 Tailwind / shadcn 的配合

- 卡片内图片：外层 `aspect-video` 或固定 `h-*`，避免图片加载前后高度跳变。
- 使用 **`object-cover`** 保持裁切一致。

---

## 3. `globals.css` 与设计令牌

`oklch` 颜色变量与 `--radius` 已定义；图片容器背景可用 **`bg-muted`** 作占位，减少「空白闪一下」的劣质感。

## 5. 小结

- **字体**：`next/font` + CSS 变量 + Tailwind = 本项目已示范。
- **图片**：`next/image` + 明确尺寸 + 远程域名白名单 = 上线必备。
- **Neon/Vercel** 不改变前端静态资源策略，但 **边缘缓存** 会让重复访问更快。
