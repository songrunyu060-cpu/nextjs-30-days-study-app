拦截路由（Intercepting Routes）虽然酷炫，但在处理“刷新保持状态”时确实显得捉襟见肘。

在真正的中后台、SaaS 或大型商用项目中，如果你追求**“刷新后弹窗不消失”且“不破坏 URL 语义”**，最主流、最稳健的方案其实是：基于 Query Parameters (查询参数) 的状态管理。

1. 行业主流方案：Query Params 控制 Modal
这种方案不使用 (.)[id] 这种复杂的目录结构，而是利用 URL 中的 ?edit=123 来控制。

URL 示例：/users?edit=123

优势：

刷新不丢失：URL 包含 edit 参数，刷新后 RSC 依然能拿到这个 ID，直接渲染弹窗。

链路简单：不需要配置 Parallel/Intercepting Routes，减少目录层级嵌套。

支持后退：用户点击“取消”或“关闭”，只需 router.push('/users')，历史记录非常清晰。
