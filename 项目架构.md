### nextjs项目目录

- 适合中小型项目

```
src/
├── app/                  # 仅仅作为 URL 的入口映射
├── features/             # 【业务核心】按功能垂直切分
│   ├── cart/             # 购物车模块
│   │   ├── components/   # 购物车特有的组件
│   │   ├── actions.ts    # 购物车相关的 Server Actions
│   │   ├── hooks.ts      # 购物车状态管理
│   │   └── types.ts      # 购物车数据结构
│   └── auth/             # 认证模块
├── components/           # 全局通用的 UI (如 Button)
└── lib/                  # 全局工具
```

- 适合大型项目

```
src/
├── app/                      # 【入口】仅定义 URL，不写具体 UI 逻辑
│   ├── dashboard/
│   │   └── page.tsx          # 引入 @/features/dashboard 中的组件
│   └── shop/
│       └── page.tsx          # 引入 @/features/shop 中的组件
├── features/                 # 【业务堡垒】按功能垂直拆分
│   ├── auth/                 # 认证功能块
│   │   ├── components/       # 登录框、注册表单
│   │   ├── actions.ts        # 登录、登出的 Server Actions
│   │   ├── types.ts          # 认证相关类型
│   │   └── schema.ts         # Zod 表单校验规则
│   ├── products/             # 商品功能块
│   │   ├── components/       # 商品列表、详情页、筛选器
│   │   ├── hooks/            # usePriceCalculator 等
│   │   └── actions.ts        # 下架、修改价格等
├── components/               # 【公共】不属于任何特定业务的组件 (如 Modal, Tooltip)
├── lib/                      # 【公共】数据库配置、常量定义
└── types/                    # 【公共】全局通用类型
```

### 目录使用规范

后端逻辑目录
- services 目录用于存放业务逻辑  直接操作数据库 Server Component可以直接调用
- server actions 目录用于存放服务器端 action (服务器函数) 需要使用 use server 修饰 用于客户端组件中
- app/api 目录用于存放 API 路由 用于外部调用

前端目录
- components 目录用于存放公共组件
- hooks 目录用于存放自定义 hooks
- lib 目录用于存放全局库函数
- types 目录用于存放全局通用类型
- config 目录用于存放全局配置
- public 目录用于存放公共资源 (图片、视频、音频等)
- styles 目录用于存放全局样式
